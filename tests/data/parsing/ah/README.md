Synthetic AH parsing fixtures for regression tests.

This directory is intended for tiny deterministic `.pcap` fixtures that exercise:
- direct IPv4 and IPv6 Authentication Header carriage using protocol / next-header `51`;
- SPI and sequence-number identity baselines for future AH-aware protocol-path work;
- outer VLAN / QinQ preservation before AH;
- IPv6 Hop-by-Hop placement before AH;
- AH tunnel-mode carriage with inner IPv4 / IPv6 payloads;
- malformed and truncated AH robustness cases.

The local helper script that generates these pcaps is intentionally local-only under `tmp/`.

## Local generation

Run from the repository root with a local Python 3 interpreter:

```bash
python3 tmp/generate_ah_pcaps.py tests/data/parsing/ah --force
```

Notes:
- the script is a local helper only and should not be treated as production tooling;
- it writes classic little-endian Ethernet `.pcap` files with deterministic MAC/IP/SPI/sequence values;
- it performs local self-validation of generated bytes before finishing;
- malformed fixtures intentionally preserve malformed AH lengths / truncation instead of auto-fixing them.

## AH basics

- IPv4 protocol number for AH: `51`
- IPv6 next-header value for AH: `51`
- AH fixed fields:
  - Next Header: 8 bits
  - Payload Length: 8 bits
  - Reserved: 16 bits
  - SPI: 32 bits
  - Sequence Number: 32 bits
- AH total length formula:
  - `(payload_length + 2) * 4`
- With the normal fixture shape here:
  - Payload Length field: `4`
  - total AH bytes: `24`
  - fixed header bytes: `12`
  - ICV bytes: `12`
- IPv6 AH total length must remain aligned to 8 bytes.
- No cryptographic ICV verification is planned for this fixture family.

## Shared constants

- Outer client MAC: `02:00:00:00:70:01`
- Outer server MAC: `02:00:00:00:70:02`
- Outer IPv4 client: `192.0.2.70`
- Outer IPv4 server: `198.51.100.70`
- Outer IPv6 client: `2001:db8:70::1`
- Outer IPv6 server: `2001:db8:70::2`
- Inner IPv4 client: `10.70.0.10`
- Inner IPv4 server: `10.70.0.20`
- Inner IPv6 client: `2001:db8:71::10`
- Inner IPv6 server: `2001:db8:71::20`
- TCP client port: `49170`
- TCP server port: `443`
- UDP client port: `53700`
- UDP server port: `443`
- Outer VLAN ID: `770`
- Outer QinQ service VLAN IDs: `771`, `772`
- Primary SPI: `0x11111111`
- Secondary SPI: `0x22222222`
- High SPI: `0xffffffff`
- Sequence values: `1`, `2`, `0xfffffffe`
- Deterministic ICV bytes:
  - `a0 a1 a2 a3 a4 a5 a6 a7 a8 a9 aa ab`

## Current implementation status

Implemented in the current branch:
- direct IPv4 AH followed by TCP or UDP;
- direct IPv6 AH followed by TCP or UDP;
- IPv6 Hop-by-Hop followed by AH and terminal UDP;
- AH tunnel-mode inner IPv4 / IPv6 flow extraction for fixtures `12`-`15`;
- SPI-based protocol-path identity:
  - `AH(spi=0x...)`
- outer VLAN / QinQ preservation before AH;
- selected-packet AH Summary / Protocol Details, including AH-owned inner IPv4 / IPv6 + TCP / UDP presentation;
- exact index-version bump for AH protocol-path serialization.

Still intentionally deferred:
- AH followed by SCTP;
- AH followed by ICMP / ICMPv6;
- nested AH chains;
- AH inside GRE, VXLAN, Geneve, GTP-U, or plain IP encapsulation;
- ICV verification / cryptographic validation.
