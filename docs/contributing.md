# Contributing

Please keep contributions small, focused, and easy to review.

- Prefer small focused commits over broad mixed changes.
- Add or update tests for domain logic when behavior changes.
- Avoid unrelated refactoring in feature or fix commits.
- Prefer clear naming over clever code.
- Document public interfaces and externally visible behavior.

If a change introduces a new public abstraction, add a short rationale in the relevant header or documentation.
